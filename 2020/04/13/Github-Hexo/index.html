一、Github-Hexo简单搭建

最近受疫情影响，在家也是无所事事，于是也就突发奇想搭建一个github-hexo个人博客。由于从没有搭建过一些东西，完全就是自学，跟着一些大佬的搭建经验和讲解、总算在不懈努力的学习下，幸运的成功无踩坑搭建了我的github-hexo链接: 个人Blog。由此，我写下这篇blog，希望能帮助大家无踩坑快速搭建自己的个人blog。

二、Hexo的安装

前面配置搭建环境，看起来不是很难。但如果你是初学者的话，还是有那么一点点小问题，不过不要气馁，我们正在一步步向成功靠近。而已经搭好的小朋友也不要高兴太早，因为困难正向你袭来。

1.创建文件夹，并安装hexo

创建的文件夹用来存放所有博客的东西。进入该文件夹目录，鼠标右击，选择Git Bash Here进入命令行。
开始安装Hexo，安装命令：npm install hexo-cli -g安装结束后，成功命令如下：

注！： 不是有部分小伙伴遇到问题了呢？下面由我来作简要解答;
如果在命令行中出现以下错误:
** bash: npm: command not found **
![](D:\blog\xizifeiyu\zifeiyu\themes\stun\source\images\icons\npm warn.png)
上网查询得知安装完nodejs之后配置windows环境变量只能保证在命令行工具中可以使用npm，如果想在git bash中使用就需要再安装一遍。
下载npm: git clone –recursive git://github.com/isaacs/npm.git

安装npm： npm install hexo-cli -g
然后就大功告成了！

2.初始化

然后初始化 命令：hexo init blog
初始化后打开，会出现下面这些文件：
node_modules：是依赖包
public：存放的是生成的页面
scaffolds：命令生成文章等的模板
source：用命令创建的各种文章
themes：主题
_config.yml：整个博客的配置信息
db.json：source解析所得到的
package.json：项目所需模块项目的配置信息
注！： 这里可能会有小伙伴初始化的时候文件存在丢失现象，例如以下错误：

这是因为文件太大了，我们只须修改文件大小即可：
npm cache clean –force
然后重新初始化 hexo init ~
如果还是不行，出现下列错误：

先运行: npm cache clean –force
然后再运行: npm install –registry=https://registry.npm.taobao.org
最后再初始化即可
提醒：初始化的时候一定要是空文件哦！

四.Github的配置

1.配置github账户信息

回到cmd中，配置github账户信息（YourName和YourEail都替换成你自己的）

2.生成SSH密钥

打开Git Bash Here命令行，输入ssh-keygen -t rsa -C “你的邮箱”然后一直回车到结束，这样就成功在C盘用户个人目录生成了.ssh文件夹。
打开.ssh文件夹，会出现两个文件，选择下方的那个文件，用记事本打开。会出现以下内容，全选复制。
登录你的gihub账号，点击Your profile

然后点击Edit profile进入，选择SSH并新建一个SSH Key
其中Title中的名称可以任意填写，将C:\Users\Administrator.ssh目录下id_rsa.pub复制的所有内容粘贴到Key中，点击Add SSH Key，SSH密钥完成。

可以在gitbash中验证是否添加成功：ssh -T git@github.com
问题：假如ssh-key配置失败，那么只要以下步骤就能完全解决

首先，清除所有的key-pair

删除你在github中的public-key
重新生成ssh密钥对

接下来正常操作
在github上添加公钥public-key:
1、首先在你的终端运行 xclip -sel c ~/.ssh/id_rsa.pub将公钥内容复制到剪切板
2、在github上添加公钥时，直接复制即可
3、保存

测试：
在终端 ssh -T git@github.com
